Learning Path

从笔记到博客的路线

01

Markdown

先把学习记录写成稳定文本，统一标题、代码块、链接和复盘格式。

02

Git

每次学习或项目改动都提交一次，保留成长轨迹，也为部署做准备。

03

HTML / CSS

理解静态页面结构、响应式布局和基础交互，掌握博客外观控制权。

04

VitePress

当文章变多后，引入静态站生成器，让 Markdown 自动变成页面。

05

GitHub Pages

把构建产物发布出去，形成一个长期可维护的公开技术履历。

Publish Checklist

发布到 GitHub Pages

这个版本是纯静态文件，上传到仓库后可以直接用 GitHub Pages 发布。以后换成 VitePress 时，仍然沿用同一套写作和部署习惯。

  1. 1

    新建 GitHub 仓库，把当前目录提交到默认分支。

  2. 2

    进入仓库 Settings / Pages，选择从分支发布。

  3. 3

    Source 选择默认分支，目录选择根目录或 docs 目录。

  4. 4

    等待 Pages 构建完成，获得公开访问地址。

Built as a static learning blog. No server, no database, just files you can own.
